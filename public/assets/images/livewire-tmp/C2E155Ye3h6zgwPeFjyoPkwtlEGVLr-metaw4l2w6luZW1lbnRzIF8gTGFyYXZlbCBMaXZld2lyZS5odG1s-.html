    Soyez incroyable sur Livewire 
 avec nos screencasts détaillés.
    
        

        Regarde maintenant
    




Les composants Livewire peuvent communiquer entre eux via un système d'événements global. Tant que deux composants Livewire vivent sur la même page, ils peuvent communiquer à l'aide d'événements et d'écouteurs.



Événements de tir



Il existe plusieurs façons de déclencher des événements à partir de composants Livewire.



Méthode A : À partir du modèle




    
<button wire:click="$emit('postAdded')">





Méthode B : à partir du composant




    
$this->emit('postAdded');





Méthode C : à partir de JavaScript global




    
<script>
    Livewire.emit('postAdded')
</script>





Écouteurs d'événements



Les écouteurs d'événement sont enregistrés dans la $listenerspropriété de vos composants Livewire.



Les écouteurs sont une paire clé->valeur où la clé est l'événement à écouter et la valeur est la méthode à appeler sur le composant.




    
class ShowPosts extends Component
{
    public $postCount;

    protected $listeners = ['postAdded' => 'incrementPostCount'];

    public function incrementPostCount()
    {
        $this->postCount = Post::count();
    }
}





Désormais, lorsqu'un autre composant de la page émet un postAddedévénement, ce composant le récupère et déclenche la incrementPostCountméthode sur lui-même.




    

        
    


    

        Si le nom de l'événement et la méthode que vous appelez correspondent, vous pouvez omettre la clé. Par exemple : protected $listeners = ['postAdded'];appellera la postAddedméthode lorsque l' postAddedévénement est émis.
    





Si vous devez nommer dynamiquement les écouteurs d'événement, vous pouvez substituer la $listenerspropriété à la getListeners()méthode protégée sur le composant :




        

                
class ShowPosts extends Component
{
    public $postCount;

    protected function getListeners()
    {
        return ['postAdded' => 'incrementPostCount'];
    }

    ...
}

    

        



Passer des paramètres



Vous pouvez également envoyer des paramètres avec une émission d'événement.




    
$this->emit('postAdded', $post->id);






        

                
class ShowPosts extends Component
{
    public $postCount;
    public $recentlyAddedPost;

    protected $listeners = ['postAdded'];

    public function postAdded(Post $post)
    {
        $this->postCount = Post::count();
        $this->recentlyAddedPost = $post;
    }
}

    

        



Evénements de cadrage



Portée aux auditeurs parents



Lorsque vous traitez des composants imbriqués , vous pouvez parfois souhaiter émettre des événements uniquement aux parents et non aux composants enfants ou frères.



Dans ces cas, vous pouvez utiliser la emitUpfonctionnalité :




    
$this->emitUp('postAdded');






    
<button wire:click="$emitUp('postAdded')">





Portée des composants par nom



Parfois, vous souhaiterez peut-être uniquement émettre un événement vers d'autres composants du même type.



Dans ces cas, vous pouvez utiliser emitTo:




    
$this->emitTo('counter', 'postAdded');






    
<button wire:click="$emitTo('counter', 'postAdded')">





(Maintenant, si le bouton est cliqué, l'événement "postAdded" ne sera émis que vers les countercomposants)



Portée sur soi



Parfois, vous souhaiterez peut-être uniquement émettre un événement sur le composant qui a déclenché l'événement.



Dans ces cas, vous pouvez utiliser emitSelf:




    
$this->emitSelf('postAdded');






    
<button wire:click="$emitSelf('postAdded')">





(Maintenant, si le bouton est cliqué, l'événement "postAdded" ne sera émis que vers l'instance du composant à partir duquel il a été émis.)



Écoute d'événements en JavaScript



Livewire vous permet d'enregistrer des écouteurs d'événement en JavaScript comme ceci :




    
<script>
Livewire.on('postAdded', postId => {
    alert('A post was added with the id of: ' + postId);
})
</script>






    

        
    


    

        Cette fonctionnalité est en fait incroyablement puissante. Par exemple, vous pouvez enregistrer un écouteur pour afficher un grille-pain (popup) dans votre application lorsque Livewire effectue certaines actions. C'est l'une des nombreuses façons de combler le fossé entre PHP et JavaScript avec Livewire.
    





Répartition des événements du navigateur



Livewire vous permet de déclencher des événements de fenêtre de navigateur comme ceci :




    
$this->dispatchBrowserEvent('name-updated', ['newName' => $value]);





Vous pouvez écouter cet événement de fenêtre avec JavaScript :




    
<script>
window.addEventListener('name-updated', event => {
    alert('Name updated to: ' + event.detail.newName);
})
</script>





AlpineJS vous permet d'écouter facilement ces événements de fenêtre dans votre code HTML :




    
<div x-data="{ open: false }" @name-updated.window="open = false">
    <!-- Modal with a Livewire name update form -->
</div>
